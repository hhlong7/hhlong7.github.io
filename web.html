Computer Architecture · RTL

RV32IMACZ — 7-Stage RISC-V CPU Pipeline

March 2026 – June 2026

A simulation-first RISC-V processor built around a seven-stage in-order core, developed in SystemVerilog. The design implements the full RV32IMAC + Zicsr/Zifencei ISA, with a two-slot dual-issue front end, branch prediction (BHT, BTB, RAS), two-level data cache hierarchy, store buffer, atomic support (LR/SC, AMOs), compressed instruction (RVC) expansion, and full CSR/trap/timer-interrupt handling.

The project evolved from a 5-stage baseline to a 7-stage pipeline (IF1 → IF2 → ID → EX1 → EX2 → MEM → WB), with each architectural addition tracked against measurable performance metrics. Software bring-up included booting FreeRTOS and running CoreMark in simulation.

GPU Architecture · RTL

LFGPU — RISC-V GPU Core

May 2026 – Present · Personal Project

My personal fork of Cal Poly CARP's GAR GPU (itself built on the open-source tiny-gpu) to learn GPU architecture hands-on. The fork already had a working round-robin warp scheduler and multi-warp execution. My goal: migrate the entire ISA from a custom encoding to standard RV32IM, enabling the GPU to run real compiler-generated kernels and targeting compatibility with the Vortex RISC-V GPU compiler.

What I've done so far

  • › Full RV32IM ISA rewrite — rearchitected instruction decoding across all RTL files (~3000 lines of SV).
  • › CSR register support — added machine-mode CSRs so kernels can read thread ID for work indexing.
  • › Assembler migration — rewrote the Rust assembler to emit standard RV32IM encodings; re-assembled all 13 kernel programs in standard RISC-V assembly.
  • › Verification — resolved a deadlock in the warp execution pipeline causing indefinite stalls; validated correctness and recorded baseline performance across all kernels in RTL simulation.

AI · Formal Reasoning · CSC 480

Limited Memory Inductive Reasoning Agent

Spring 2026 · Team: Christian Moran, Yasmin Akkaya, Long Ho, Dennis Kulik

An AI agent that ingests a stream of facts under a fixed memory budget and must decide what to remember, what to forget, and when to generalize repeated observations into first-order logic (Horn clause) rules — so it can correctly answer queries about facts it never stored.

The core insight: a FOL Compression Agent replaces repeated facts with reusable rules via Z3 formal reasoning, then evicts derivable facts from memory. On chain datasets, baselines answer "Unknown" for non-adjacent transitive pairs; the FOL agent commits a transitivity rule and derives them. Benchmarked against three baseline agents (recent, random, importance) across 8 fixed datasets spanning greater-than, divisibility, equality, and mixed relations.
